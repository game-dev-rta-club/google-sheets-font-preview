# Store page review

This document captures the current positioning and improvement plan for the Google Workspace Marketplace store page.

## Current positioning

Font Preview Sidebar should be presented as a visual review tool for spreadsheet-managed text.

Core message:

```text
Review spreadsheet text as it will actually appear.
```

The app is strongest when it is framed around the problem that spreadsheet text is easy to manage but hard to judge visually. The store page should make it clear that the sidebar helps teams review, edit, and fit localized text before it ships.

## Target users

- Localization and translation reviewers who manage text in spreadsheets.
- App, web, and game teams that need to check whether text fits in real UI space.
- Copy QA teams that review large amounts of text with context.

## Recommended listing copy

Use the copy in [marketplace-listing.md](marketplace-listing.md) as the current source of truth for Marketplace fields.

Short description:

```text
Review, edit, and fit localized text in Google Sheets™ with real fonts, screenshots, and notes.
```

Post-installation hint:

```text
Open or reload a spreadsheet, then choose Extensions > Localization > Font Preview. Use the Help button to create a sample sheet.
```

## Screenshot review

Current screenshots are accepted-size Marketplace assets and explain the feature set, but the first screenshot can be stronger as the storefront hero image.

### Priority 1: first screenshot

Goal:

```text
Show that spreadsheet text can be reviewed visually before it ships.
```

Recommended title:

```text
Review spreadsheet text as it will actually appear
```

Recommended direction:

- Show a realistic Google Sheets™ spreadsheet on the left and the sidebar on the right.
- Keep the selected row and sidebar preview visually connected.
- Reduce large red arrows and annotation-heavy overlays.
- Prefer a product screenshot feeling over a tutorial-slide feeling.
- Keep text large enough to read in the Marketplace carousel.

### Priority 2: supporting screenshots

The remaining screenshots can continue to focus on:

- local font drag and drop
- real-time editing and size-aware preview
- screenshot and note context

If these are updated later, keep each screenshot focused on one user benefit rather than showing every feature at once.

## Review checklist before changing the store page

- Does the first screen explain the problem within a few seconds?
- Does the short description mention the main user value, not only features?
- Does the detailed description explain who the tool is for?
- Does the post-installation hint match the actual Google Sheets™ menu path?
- Are Google product names marked with `™` where required?
- Does at least one screenshot show the app inside a realistic spreadsheet workflow?
- Is it clear that the app is free and does not require a paid subscription?

## Success signals to watch

Google Workspace Marketplace does not expose a full public conversion funnel, so use available signals:

- installs after listing copy or screenshot changes
- user reviews and review text
- GitHub issues or support questions about installation
- repeated confusion around what the app does or how to open it

When possible, change one major store-page element at a time so the effect is easier to understand.
